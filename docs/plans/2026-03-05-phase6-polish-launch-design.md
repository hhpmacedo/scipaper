# Phase 6: Polish & Launch — Design Document

**Date:** 2026-03-05
**Status:** Approved

## Goal

Make Signal ready for public subscribers: branded landing page with subscribe form, welcome email, and real production URL.

## Architecture

Minimal code approach leveraging Buttondown for subscriber management, welcome email delivery, and analytics. The only new code is a landing page generator added to the existing web archive module. Welcome email copy lives in the repo but is configured manually in Buttondown's dashboard.

## Decisions

- **Landing page:** Static HTML generated by `generate_landing_page()` in `web.py`, with embedded Buttondown subscribe form
- **Welcome email:** Single email, content drafted in `docs/welcome-email.md`, configured manually in Buttondown Automations
- **Analytics:** Buttondown built-in only (open rates, clicks, subscriber growth)
- **Domain:** `signal.hugohmacedo.com`

## Changes

### 1. Landing Page (`scipaper/publish/web.py`)

Add `generate_landing_page(editions, config)` that produces `index.html` with:

- Hero: "Signal" branding + "AI Research for the Curious" tagline
- Value proposition: 3 bullet points (what, frequency, audience)
- Buttondown subscribe form: plain HTML `<form>` posting to `https://buttondown.com/api/emails/embed-subscribe/{username}`
- Latest edition preview: link to most recent edition + lead piece title
- Footer: archive link, RSS link

### 2. Archive Restructure

- Current `generate_index_page()` output moves from `index.html` to `archive.html`
- `generate_web_archive()` updated to produce both landing page and archive
- Navigation updated: landing page → archive, archive → landing page, edition pages → both

### 3. WebConfig Update

Add `buttondown_username: str` field to `WebConfig` for the subscribe form action URL.

### 4. URL Updates

| Location                               | Old Default                  | New Default                      |
| -------------------------------------- | ---------------------------- | -------------------------------- |
| `WebConfig.site_url`                   | `https://signal.example.com` | `https://signal.hugohmacedo.com` |
| `PipelineConfig.web_base_url`          | `https://signal.example.com` | `https://signal.hugohmacedo.com` |
| `.github/workflows/weekly-edition.yml` | `https://signal.example.com` | `https://signal.hugohmacedo.com` |

### 5. Welcome Email (`docs/welcome-email.md`)

Markdown file with welcome email copy. Hugo configures this manually in Buttondown Automations dashboard.

### 6. Tests

- Landing page: valid HTML, subscribe form present, latest edition link, nav links
- Archive page: renamed from index, still lists all editions
- Navigation: cross-links between landing, archive, and edition pages
- Subscribe form: correct Buttondown action URL with configured username

## Out of Scope

- Analytics code (Buttondown handles it)
- Welcome email sending code (Buttondown Automations)
- Deployment automation (manual/separate)
- Subscriber management (Buttondown handles it)
