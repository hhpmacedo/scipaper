# Pipeline Versioning

Explicit version history of the two core algorithms: curation (paper discovery and selection) and writing (content generation and verification). Updated whenever either algorithm changes.

## Current Live Version (TL;DR)

### Curation (v3.1)

Fetches up to 200 papers from ArXiv (last 7 days), enriches them concurrently via Semantic Scholar (citations, references) and Hacker News (social signal). Scores each paper on a five-factor weighted relevance axis (35% topic match, 20% keyword match, 15% institution, 15% citation velocity, 15% social signals) using a weekly anchor document (`data/anchors/*.yaml`) that defines hot topics, boost keywords, and declining topics (30% penalty). Papers matching `declining_topics` are penalized. A second LLM pass (Sonnet) scores narrative potential on the top candidates. The two scores combine 50/50 into a composite. Greedy selection picks 8 papers (min 3, max 10) with diversity caps (max 2 per institution, max 2 per topic), relaxing constraints if minimum not met. Runners-up (relevance >= 4.0) feed Quick Takes. A two-pass architecture exists (LLM only for top 20, heuristic for rest) but is not wired as the default path.

### Writing (v4.2)

Opus generates each article from the paper's full text (preprocessed via `prepare_text_for_llm()` to strip references/appendix and truncate at sentence boundaries). The generation prompt enforces citation grounding -- every factual claim must cite `[section X.Y]`, `[Abstract]`, `[Table N]`, or `[Figure N]`. Output is structured JSON: title, hook, structured abstract (What They Did / Key Result / Why It Matters), content body, section labels, and optional hero figure number. The prompt includes editorial rules: no redundancy between hook and content, varied endings (not always a caveat), and practitioner-focused depth over mechanism detail. Word target: 800-1200. A second LLM pass (Sonnet) verifies each claim against the cited passage, classifying issues by severity; any critical issue or 3+ major issues rejects the piece. A rule-based style checker enforces banned words, required structure, word count, and citation density per the Style Constitution. Quick Takes for runners-up use 2-sentence abstract extraction (no LLM). Output is delivered as Buttondown email draft (lead piece full, others preview + link) and a static brutalist web archive with TOC, hero figures, and Signal boxes.

| -------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **v1.0** | **Initial weighted scoring.** Five-factor relevance (35% topic, 20% keyword, 15% institution, 15% citation velocity, 15% social signals) + LLM narrative potential, 50/50 composite. Select 5 papers, diversity caps (max 2 same institution/topic). Anchor doc defines hot/declining topics.             |
| **v1.1** | **Reliability hardening.** Added `@api_retry` with exponential backoff (1-8s, 3 attempts) for all external API calls. No algorithm change.                                                                                                                                                                |
| **v1.2** | **Model centralization.** LLM model version extracted to single config source of truth instead of hardcoded per-module. No algorithm change.                                                                                                                                                              |
| **v1.3** | **Concurrent enrichment.** Semantic Scholar (semaphore 10) and HN (semaphore 5) enrichment now parallel instead of sequential. No algorithm change.                                                                                                                                                       |
| **v2.0** | **DEC-004 selection buffer.** Target raised from 5 to 8 papers, max from 8 to 10, to buffer against verification-stage failures. Scoring unchanged.                                                                                                                                                       |
| **v3.0** | **Two-pass scoring architecture.** Pass 1: pure-Python relevance score on all papers (instant, no API). Pass 2: LLM narrative scoring only on top 20 by relevance; remainder get heuristic narrative score (keyword bonuses for "surprising", "practical", "open-source", etc.). Saves ~80% of LLM calls. |
| **v3.1** | **Pipeline wiring simplification.** Two-pass available but pipeline.py reverted to single-pass `score_papers()` as default entry point.                                                                                                                                                                   |

## Writing Pipeline

| Version  | Changes / Thesis                                                                                                                                                                                                                                                                                                                                                             |

| **v1.0** | **Citation-grounded generation.** GENERATION_SYSTEM_PROMPT requires every claim cite `[section X.Y]`, `[Abstract]`, `[Table N]`, or `[Figure N]`. Quick Takes generated by LLM. |

| **v1.3** | **Model upgrade for generation.** Generation switched from Sonnet to Opus for higher prose quality. Verification/scoring stay on Sonnet for speed/cost.                                                                                                                                                                                                                      |
| **v2.0** | **Drop LLM Quick Takes.** Replaced LLM-generated quick takes with abstract extraction (first 2 sentences). Removed QUICK_TAKE_PROMPT. Simpler, cheaper, more predictable.                                                                                                                                                                                                    |
| **v2.1** | **Buttondown email + hybrid layout.** Replaced multi-provider email with Buttondown API (creates draft). Lead piece rendered in full; secondary pieces show hook + first paragraph + "Read more" link.                                                                                                                                                                       |
| **v3.0** | **Hero figures.** Generation prompt now selects most impactful figure from paper. Piece model gains `hero_figure_url`/`hero_figure_caption`. Rendered after hook in both email and web.                                                                                                                                                                                      |
| **v3.1** | **Structured abstract (Signal box).** Generation outputs 3-part summary: What They Did / Key Result / Why It Matters. Rendered as a distinct box before article content in both email and web.                                                                                                                                                                               |
| **v4.0** | **Editorial quality overhaul.** Major prompt revision: AVOIDING REDUNDANCY (don't restate hook in content), STYLE VARIATION (not every piece ends with caveat), TECHNICAL DEPTH (prioritize practitioner "so what" over mechanism). Quick takes changed to 2-sentence format.                                                                                                |
| **v4.1** | **Hook restoration.** Reverted v4.0's hook suppression -- editorial feedback said the hook "is the best part". Redundancy now handled purely by prompt instruction, not rendering logic.                                                                                                                                                                                     |
